# deliverables/

Final, presentation-ready outputs — the work as it's meant to be seen.

This is the front-of-house folder. Everything here is polished and self-contained: the version I'd hand to an executive committee, a recruiter, or the instructor without a walkthrough. The reasoning, drafts, and working files stay upstream in `docs/`, `models/`, and `analysis/`; what lands here is the finished cut.

## Contents (Stage 5)

- `{date}-{lastname}-{company}-llm-raw.md` — the unedited LLM output produced by executing the Stage 4 spec in a clean-context session. The data input for the evaluation work, not the deliverable itself.
- `{date}-{lastname}-{company}-final-analysis.md` — the evaluated, corrected, annotated final analysis with my executive judgment.
- `{date}-{lastname}-{company}-spec-retrospective.md` — structured self-evaluation of my Stage 4 spec against the actual Stage 5 LLM output.
- `prompt-log.md` — running log of meaningful AI prompts and outputs across all stages, with per-session editorial-control notes.
