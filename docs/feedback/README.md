# docs/feedback/

Instructor review files received via pull request.

Each file is the markdown body of a PR the instructor opened against this repo — feedback only, no scores. The original PRs were merged so the reviews live permanently on `main` rather than only as closed PR threads.

## Naming convention

`stage{N}-review-YYYY-MM-DD.md` — matches the file the instructor's PR creates.

## How I respond to feedback

Selective incorporation is documented in `deliverables/prompt-log.md` (dated rows) and, where the suggestions affect the spec itself, in `deliverables/{date}-{lastname}-{company}-spec-retrospective.md`. Disagreements with feedback, if any, are noted in the same places — silence on a suggestion would read as oversight rather than judgment.
