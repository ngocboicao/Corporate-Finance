# docs/

Everything written rather than calculated lives here — the words, not the numbers.

I treat this as the narrative layer of the repo: memos that explain decisions, specs that pin down how a model works, and plans that track where the work is headed. The spreadsheets sit in `models/`; the polished final outputs sit in `deliverables/`. This folder is the reasoning in between.

The subfolders keep types of writing from bleeding into each other:

- `decisions/` — why a choice was made
- `specs/` — how something is built
- `plans/` — what's being done and by when
- `templates/` — where the canonical course templates live (pointer only)
