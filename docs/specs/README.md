# docs/specs/

Technical specifications for the financial models (Stage 4).

A spec here does for a model what a design spec does for a product: it states the inputs, the assumptions, the outputs, and the logic connecting them, so the result can be rebuilt by someone who wasn't in the room. If a formula choice isn't obvious, this is where it gets explained. The goal is reproducibility — open the spec, follow it, land on the same numbers.
